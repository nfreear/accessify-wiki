This is a prototype for a bookmarklet and service that will apply accessibility fixes to third-party sites.

Accessify (*) bookmarklet – right-click and add or save me to your browser's bookmarks or favourites bar (Chrome and Safari users will wish to start by copying the link). 




 1. Fixes are authored in a human-readable format, for example, YAML (*),
    //gist.github.com/..#file-baidu-search-wai-aria-en-yaml - perhaps via a Wiki,

 2. The fixes are submitted to a system which binds a URL pattern, for example,
    `baidu.com/*` to the fixes,

 3. A user visits a page for which the system has fixes, for example,
    //baidu.com/s?word=...,

 4. The browser plugin or bookmarklet queries the system for fixes,
    which are delivered - currently represented by,
    //views.scraperwiki.com/run/yaml_2..?..url=..ouice-wai-aria.yaml (*),

 5. The plugin or bookmarklet runs AccessifyHTML5.js to insert the fixes in the page,

 6. And, the user experiences the page with fixes - try Juicy Studio toolbar, YAML Debug.



Notes:

 (*) The name "Accessify" is only provisional at this stage,

 * The location of the bookmarklet Javascript will change!

 * The system would be designed to complement the important work of Fix the Web, RNIB and others,

 * However, is there a danger, as alluded to in the Fix The Web FAQ?

    “Won’t website owners think they don‘t need to check [or fix] web-accessibility
     issues now because [insert name] will do this work for them?”
